# File Service — Overview

## Purpose

The File Service is the core of FileNest Phase 1. It handles:

- **Upload initiation** — creates a file record in the database and returns a presigned URL the client uses to PUT bytes directly to object storage (MinIO in dev, S3 in production).
- **File metadata retrieval** — fetch a single file's full metadata by ID.
- **File listing** — cursor-paginated list of files within a project, optionally scoped to a folder.

The service deliberately does NOT handle the byte transfer itself. Clients upload directly to storage, which removes the service from the data path and allows unbounded file sizes.

## Running locally

```bash
just dev     # start docker compose (postgres, redis, minio, nats, clamav)
just migrate # apply database migrations
just file    # start file service on http://localhost:8001 with hot-reload
```

API docs (auto-generated by FastAPI): http://localhost:8001/docs

## Port & prefix

| | Value |
|-|-------|
| Default port | `8001` |
| Route prefix | `/v1` |
| Full base URL (local) | `http://localhost:8001/v1` |

## Internal structure

```
services/file/
├── main.py         — app factory, error handler, lifespan
├── router.py       — HTTP route handlers (thin — delegate to service)
├── service.py      — all business logic
├── repository.py   — all DB access (FileRecord ORM + queries)
├── schemas.py      — Pydantic request/response models
├── dependencies.py — FastAPI Depends() wiring
└── docs/           — this folder
```

## Dependencies (Phase 1)

| Dependency | Used for |
|-----------|---------|
| PostgreSQL | Storing file metadata in the `files` table |
| MinIO / S3 | Object storage (presigned URL target) |
| Redis | (Planned) rate limiting, idempotency keys |
| NATS | (Planned Phase 2) processing pipeline trigger via outbox |

## What this service does NOT do (yet)

- Virus scanning (Phase 2 — ClamAV integration)
- Multipart upload (Phase 2)
- MIME type validation (Phase 2)
- Processing pipeline orchestration (Phase 2)
- Download URL generation / signed download links (Phase 2)
- Folder management (Phase 3)
- Metadata schema validation (Phase 3)
