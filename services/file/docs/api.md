# File Service — API Usage Guide

> The complete OpenAPI schema with all request/response shapes is auto-generated by FastAPI and available at **http://localhost:8001/docs** when the service is running. This document covers usage patterns, authentication, and common examples.

## Authentication

All endpoints require a Bearer token in the `Authorization` header:

```
Authorization: Bearer fn_live_<key>
```

| Token type | Format | Use |
|-----------|--------|-----|
| Live API key | `fn_live_…` | Server-to-server calls |
| Test API key | `fn_test_…` | CI / test environments |
| Upload token | `fn_upload_token_…` | Short-lived browser token (Phase 2) |

## Endpoints

### POST /v1/files/upload

Initiate a single-file upload. Returns a presigned URL for the client to PUT bytes to.

**Required scope:** `files:upload`

**Request:**
```json
{
  "filename": "report.pdf",
  "content_type": "application/pdf",
  "size_bytes": 2097152,
  "folder_id": null,
  "metadata": {
    "source": "mobile-app",
    "patient_id": "p_123"
  }
}
```

**Response 201:**
```json
{
  "file_id": "018e1c2f-4a3b-7d9e-b123-456789abcdef",
  "upload_url": "https://s3.amazonaws.com/filenest/018e1c2f-...?X-Amz-Signature=...",
  "upload_id": null,
  "expires_at": "2026-06-17T13:00:00Z"
}
```

After receiving this response, the client PUTs the file bytes directly to `upload_url`:
```
PUT <upload_url>
Content-Type: application/pdf
[binary file bytes]
```

### GET /v1/files/{file_id}

Fetch the full metadata record for a single file.

**Required scope:** `files:read`

**Response 200:**
```json
{
  "id": "018e1c2f-4a3b-7d9e-b123-456789abcdef",
  "organization_id": "org_abc",
  "project_id": "proj_xyz",
  "filename": "report.pdf",
  "content_type": "application/pdf",
  "size_bytes": 2097152,
  "status": "ready",
  "storage_key": "org_abc/proj_xyz/018e1c2f-...",
  "folder_id": null,
  "metadata": { "source": "mobile-app" },
  "created_at": "2026-06-17T12:00:00Z",
  "updated_at": "2026-06-17T12:01:30Z"
}
```

**Response 404:** File does not exist or is outside the caller's project scope.

---

### GET /v1/files

Return a cursor-paginated list of files in the current project.

**Required scope:** `files:read`

**Query parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `folder_id` | string \| null | null | Filter to files in a specific folder |
| `limit` | int | 50 | Page size (1–200) |
| `cursor` | string \| null | null | Last `id` from previous page |

**Response 200:**
```json
{
  "items": [ ...FileResponse objects... ],
  "total": 50,
  "cursor": "018e1c30-0000-0000-0000-000000000000"
}
```

When `cursor` is `null` in the response, you have reached the last page.

**Fetching subsequent pages:**
```
GET /v1/files?limit=50&cursor=018e1c30-0000-0000-0000-000000000000
```

## Error responses

All errors follow the standard envelope:
```json
{
  "error": "NOT_FOUND",
  "message": "File 018e1c2f not found",
  "detail": {}
}
```

| HTTP status | `error` code | Cause |
|-------------|-------------|-------|
| 401 | `MISSING_CREDENTIALS` | No Authorization header |
| 401 | `INVALID_TOKEN` | Token is expired or invalid |
| 403 | `PERMISSION_DENIED` | Token lacks required scope |
| 404 | `NOT_FOUND` | File not found in this project |
| 413 | `FILE_TOO_LARGE` | Exceeds multipart threshold (Phase 2) |
| 422 | `VALIDATION_ERROR` | Business-rule violation |
| 422 | (FastAPI default) | Invalid request body shape |
