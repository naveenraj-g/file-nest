# Files API

## Upload a file

```
POST /v1/projects/{project_id}/files/upload
```

**Required scope:** `files:upload`  
**Content-Type:** `multipart/form-data`

```bash
curl -X POST https://api.filenest.io/v1/projects/{id}/files/upload \
  -H "Authorization: Bearer fn_live_..." \
  -F "file=@invoice.pdf" \
  -F "filename=invoice.pdf"
```

**Response `201`:**

```json
{
  "id": "file_01j...",
  "filename": "invoice.pdf",
  "size": 204800,
  "mime_type": "application/pdf",
  "status": "processing",
  "storage_key": "proj_01j.../invoice.pdf",
  "created_at": "2026-06-20T10:00:00Z"
}
```

The file moves to `status: "ready"` once the processing pipeline completes (typically under 5 seconds for small files).

## List files

```
GET /v1/projects/{project_id}/files
```

**Required scope:** `files:read`

**Query parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `page` | `number` | Page number (default: 1) |
| `page_size` | `number` | Items per page (default: 50, max: 200) |
| `status` | `string` | Filter by status (`ready`, `processing`, etc.) |
| `mime_type` | `string` | Filter by MIME type prefix |

## Get a file

```
GET /v1/projects/{project_id}/files/{file_id}
```

**Required scope:** `files:read`

## Download a file

```
GET /v1/projects/{project_id}/files/{file_id}/download
```

**Required scope:** `files:download`

Returns a `302` redirect to a 15-minute presigned download URL. The download URL is signed for the specific storage backend — no FileNest auth is required to fetch it.

## Delete a file

```
DELETE /v1/projects/{project_id}/files/{file_id}
```

**Required scope:** `files:delete`

Soft-deletes the file by setting `deleted_at`. The storage object is removed by a background job.

## Error codes

| Status | Code | Meaning |
|--------|------|---------|
| `404` | `file_not_found` | File does not exist or is deleted |
| `409` | `file_quarantined` | File failed virus scan — download blocked |
| `413` | `file_too_large` | File exceeds the project's max upload size |
