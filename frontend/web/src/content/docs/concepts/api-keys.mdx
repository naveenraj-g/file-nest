# API Keys

API keys authenticate server-to-server requests to the FileNest API. They are managed by the FileNest IAM and scoped to an organization.

## Key prefixes

| Prefix | Use |
|--------|-----|
| `fn_live_...` | Production environment |
| `fn_test_...` | Test / sandbox environment |

<Callout type="warning" title="Keep keys secret">
API keys are long-lived credentials. Never commit them to source control or expose them in client-side code.
</Callout>

## Scopes

Every API key carries a set of scopes that restrict what it can do:

| Scope | Description |
|-------|-------------|
| `files:upload` | Upload files |
| `files:download` | Generate download URLs |
| `files:read` | List and get file metadata |
| `files:delete` | Delete files |
| `files:update_metadata` | Update file metadata |
| `projects:read` | Read project configuration |
| `projects:update` | Update project settings |
| `api_keys:create` | Create new API keys |
| `api_keys:revoke` | Revoke API keys |
| `audit:read` | Read audit logs |
| `compliance:manage` | Manage WORM, legal hold, retention (Phase 8) |

## Generating a key

API keys are generated in the Console under **API Keys** inside a project, or during the onboarding wizard. The full key value is shown exactly once — save it immediately.

## Using a key

Pass the key as a Bearer token in the `Authorization` header:

```bash
curl https://api.filenest.io/v1/projects/{id}/files \
  -H "Authorization: Bearer fn_live_abc123..."
```

## Upload tokens

For browser-side uploads, use short-lived **upload tokens** (`fn_upload_token_...`) instead of API keys. These are generated by your server and issued to the browser through a token endpoint. See [React SDK](/docs/sdks/react) for the setup.

## Revoking a key

Revoked keys are immediately rejected. Revocation is permanent — you cannot restore a revoked key.
