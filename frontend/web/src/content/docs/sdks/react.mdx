# React SDK

The `@filenest/react` SDK provides ready-made upload components and data hooks that work with any React application. The API key never reaches the browser — the SDK exchanges short-lived upload tokens through your own token endpoint.

## Installation

<PackageTabs pkg="@filenest/react" />

## Provider setup

Wrap your app (or subtree) with `FileNestProvider`. It fetches and caches upload tokens automatically.

```tsx
import { FileNestProvider } from "@filenest/react";

export default function RootLayout({ children }) {
  return (
    <FileNestProvider
      tokenEndpoint="/api/filenest-token"
      projectId={process.env.NEXT_PUBLIC_FILENEST_PROJECT_ID}
    >
      {children}
    </FileNestProvider>
  );
}
```

You must implement `tokenEndpoint` on your server. See the [Next.js SDK](/docs/sdks/nextjs#upload-token-endpoint) for a reference implementation.

## Upload components

### FileUpload

```tsx
import { FileUpload } from "@filenest/react";

// Drag-and-drop zone
<FileUpload
  variant="dropzone"
  accept={["image/*", "application/pdf"]}
  maxSize={50 * 1024 * 1024}
  onComplete={(files) => console.log("Uploaded:", files)}
  onError={(err) => console.error(err.message)}
/>

// Click-to-upload button
<FileUpload
  variant="button"
  accept={["image/*"]}
  onComplete={(files) => console.log("Uploaded:", files)}
/>
```

### FilePreview

Inline preview — images render directly, PDFs via iframe.

```tsx
import { FilePreview } from "@filenest/react";

<FilePreview fileId="file_01j..." />
```

### FileViewer

Full-page viewer with toolbar and download button.

```tsx
import { FileViewer } from "@filenest/react";

<FileViewer fileId="file_01j..." />
```

## Upload hooks

### useUpload

Programmatic upload — trigger via code, track per-file progress.

```tsx
import { useUpload } from "@filenest/react";

function UploadButton() {
  const { upload, uploads, isUploading, cancel, retry, clear } = useUpload({
    onComplete: (file) => console.log("Done:", file.id),
    onError: (err) => console.error(err.message),
  });

  return (
    <>
      <input
        type="file"
        multiple
        onChange={(e) => upload(Array.from(e.target.files ?? []))}
      />
      {uploads.map((u) => (
        <div key={u.id}>
          {u.filename}: {u.progress}%
          {u.status === "failed" && <button onClick={() => retry(u.id)}>Retry</button>}
        </div>
      ))}
    </>
  );
}
```

## Data hooks

### useFiles

Paginated file list backed by TanStack Query.

```tsx
import { useFiles } from "@filenest/react";

const { files, totalCount, hasMore, isLoading, loadMore, refresh } = useFiles({
  status: "ready",
  tags: ["finance"],
  limit: 20,
});
```

### useInfiniteFiles

Infinite-scroll variant.

```tsx
import { useInfiniteFiles } from "@filenest/react";

const { files, hasMore, fetchMore, refresh } = useInfiniteFiles({ status: "ready" });
```

### useFile

Single file detail with automatic revalidation.

```tsx
import { useFile } from "@filenest/react";

const { file, isLoading, isError, mutate } = useFile("file_01j...");
```

### useFolder

Folder navigation with breadcrumb trail and subfolder listing.

```tsx
import { useFolder } from "@filenest/react";

const { folder, files, subfolders, isLoading, breadcrumbs } = useFolder(folderId);
```

`folderId` can be `null` to show the project root. Update it to navigate into a subfolder — breadcrumbs update automatically.

### useSearch

Debounced full-text search.

```tsx
import { useSearch } from "@filenest/react";

const { results, facets, isLoading, totalCount, search, hasMore } = useSearch({
  debounceMs: 300,
  limit: 20,
});

// Trigger a search
search("invoice Q1 2026");
```
